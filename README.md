# DanceGuard-AI

## 项目简介
DanceGuard-AI 是一个面向新疆民族舞非遗传承场景的智能评估系统，聚焦 **黑走马（Kara Jorga）** 与 **木卡姆（Muqam）** 两类舞蹈动作的数字化建模、实时评估与离线分析。项目基于 **华为昇思 MindSpore** 构建舞蹈评分模型，以 **MoCap / BVH 骨骼序列** 为主导输入，同时兼容浏览器视频展示、Rebocap 实时骨骼流、离线 `NPY/BVH` 导入评估等多种使用方式。

本项目对应第十届华为 ICT 大赛中国创新赛作品：
- 作品名称：**舞彩边疆——面向舞蹈传承的 AI 保护方案**
- 学校：**新疆大学**
- 队伍：**Xju第一少年团**
- 赛题：**赛题1：基于昇思 MindSpore 打造 AI 创新应用**

## 背景与价值
新疆地区拥有丰富的舞蹈类非物质文化遗产资源，但传统的展示和教学形式存在以下问题：
- 游客与学习者大多停留在“被动观看”，交互深度不足。
- 专业舞蹈教师与驻场演员成本较高，难以大规模复制。
- 标准动作难以数字化保存，优秀舞蹈动作经验难以长期沉淀。

DanceGuard-AI 的目标不是单纯做一个动作识别 Demo，而是构建一套可落地的民族舞智能评估系统：
- 将标准舞蹈动作转化为可计算、可比对、可评分的数字资产。
- 在浏览器端提供现代化可视化工作台，提升展示与使用体验。
- 通过实时评分、弱项分析、历史记录和离线评估，形成完整训练闭环。

## 核心能力
### 1. 实时评估
- 浏览器直接接入 OBS / 云景 / 虚拟摄像头用于视频展示。
- 后端接入 Rebocap VMC 骨骼流进行实时评分。
- 实时输出：
  - 当前得分
  - 判定等级（PERFECT / GREAT / GOOD / WARN）
  - 连击统计
  - 最弱关节提示
  - 战报流与评分曲线

### 2. 离线导入评估
- 支持上传 `NPY` 动作序列进行离线评估。
- 支持上传 `BVH` 文件并解析后进入评估链路。
- 自动写入历史记录，形成可回放的评估结果。

### 3. 多舞种管理
- 当前支持舞种：
  - 黑走马（Kara Jorga）
  - 木卡姆（Muqam）
- 前端可切换舞种。
- 后端保留多模型 / 多标准动作扩展接口。

### 4. 结果分析与记录管理
- 历史记录查看与筛选。
- 历史详情页展示：
  - 平均分
  - 最高连击
  - 时长
  - 最弱关节
  - 关节级分析
  - 时间片段分析
  - 原始文件入口
- 支持历史重评、删除、摘要导出。

### 5. 游戏化反馈与展示
- Apple 风格网页仪表盘。
- 图标化侧边栏、卡片化信息布局、毛玻璃与弥散背景。
- 判定流、连击、实时曲线、设备状态等信息统一收口到一个工作台内。

## 技术路线
### 算法与模型
- 框架：**MindSpore 2.6+ / 2.8.0**
- 数据主模态：**MoCap / BVH 骨骼序列**
- 设计思想：**MoCap 主导，多模态保留拓扑，视觉模态可优雅降级**
- 已实现模块包括：
  - 合成数据生成
  - 评分网络训练与推理
  - BVH 解析
  - VMC 实时骨骼接收
  - 实时评分运行时

### 系统工程
- 前端：原生 HTML / CSS / JavaScript
- 后端：Python HTTP 服务
- 数据存储：本地 SQLite
- 实时通信：SSE（Server-Sent Events）
- 视频展示：浏览器 `getUserMedia`
- 骨骼流输入：Rebocap VMC

## 快速开始
### 1. 启动项目
Windows 下可直接双击：
```bat
start.bat
```

或命令行启动：
```powershell
.venv\Scripts\python.exe main.py
```

启动后访问：
```text
http://127.0.0.1:8000
```

### 2. 实时评估链路
1. 在 OBS 中启动虚拟摄像头。
2. 在 Rebocap 侧开启 VMC 输出。
3. 打开网页工作台。
4. 选择舞种与视频设备。
5. 点击“开始评估”。

### 3. 离线评估链路
1. 打开“离线导入”页面。
2. 上传 `NPY` 或 `BVH` 文件。
3. 查看导入结果与历史记录。

## 部署说明
### 方式一：本地快速部署
适合项目演示、答辩和本地联调。

1. 克隆仓库：
```powershell
git clone https://github.com/deafenken/DanceGuard-AI.git
cd DanceGuard-AI
```

2. 准备 Python 3.9 环境并安装依赖。

3. 启动 Web 服务：
```powershell
python main.py
```

4. 浏览器访问：
```text
http://127.0.0.1:8000
```

### 方式二：手动服务部署
适合在比赛现场设备或演示电脑上稳定运行。

1. 准备独立 Python 环境。
2. 安装 MindSpore 与必要依赖。
3. 确保以下资源可用：
   - 标准动作数据
   - 模型权重文件
   - OBS 虚拟摄像头
   - Rebocap VMC 输出
4. 使用命令启动：
```powershell
python main.py
```
5. 如需局域网访问，可在 `web/server.py` 中调整监听地址与端口。

### 部署注意事项
- 浏览器端视频展示依赖摄像头权限。
- 后端评分依赖 Rebocap VMC 输入，不应只依赖视频流。
- 模型权重、运行数据库、导入缓存、录制结果默认不进入 Git 仓库。
- 若用于公开展示，请补充真实权重与合法授权的数据文件。
